# 아정당 운영 추천 대시보드

> Snowflake Hackathon 2026 Korea — 테크/개발 트랙

통신·렌탈 플랫폼 **아정당**의 CEO 및 마케팅 담당자를 위한 **AI 기반 운영 전략 추천 대시보드**입니다.
내부 계약 데이터(V01~V11)와 외부 신호(날씨·공휴일·뉴스)를 종합하여, 지금 가장 먼저 점검해야 할 지역·상품·채널을 추천합니다.

---

## 주요 기능

| 탭 | 기능 | 설명 |
|----|------|------|
| 1. 사업 현황 | KPI 대시보드 & 퍼널 분석 | 선택 기간의 계약·완료율 추이, 인터넷/렌탈 카테고리별 성과, 전환 퍼널 |
| 2. 문제 진단 | 렌탈 상품·지역·채널 진단 | 완료율 하위 상품/지역 식별, 문제 유형 자동 분류(매출 영향·이탈 리스크·긴급) |
| 3. 전략 추천 | Cortex AI 전략 도출 | 5개 카테고리(매출긴급/이탈방지/성장기회/효율개선/외부대응)별 우선순위 전략 |
| 4. 시뮬레이션 | What-if 시나리오 분석 | 폭염·이사시즌 등 시나리오별 보수적/기본/공격적 시뮬레이션 + AI 대응 전략 |
| 5. AI 질의 | 자유 질의응답 | 대시보드 데이터 기반으로 Cortex AI에게 전략적 질문 |

## AI 전략 추천 프로세스

전략 추천은 **3-Layer 컨텍스트 시스템**으로 동작합니다:

1. **Strategy Context (연간 실적)** — 전년 대비 계약·완료율 변화, 카테고리별 성과
2. **Trend Context (월별 추세)** — 최근 6개월 모멘텀, 방향성(악화 가속/소폭 하락/소폭 개선/개선 가속), 급변 경고
3. **Data Facts (상세 데이터)** — 지역×카테고리, 렌탈 상품별, 채널, 콜센터, 외부 신호

이 3개 레이어를 프롬프트에 결합하여 Cortex AI가 5개 카테고리별 우선순위 전략을 도출합니다.

## 성능 최적화

- **SQL 캐싱**: `@st.cache_data(ttl=3600)` — 동일 세션 내 SQL 재실행 방지
- **DataFrame 해싱 회피**: 파생 데이터는 `st.session_state`에 캐시 (pickle 해싱 오버헤드 제거)
- **Lazy AI 호출**: 모든 Cortex AI 분석은 버튼 클릭 시에만 실행 (탭 전환 시 0건의 AI 호출)
- **낙관적 Cortex 체크**: 초기 ping 제거, 실제 호출 시 에러 감지
- **날짜 변경 감지**: 기준일 변경 시 관련 캐시 자동 클리어
